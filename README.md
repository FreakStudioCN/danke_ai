## 🌟 项目简介

**壳生同频**是一个基于 MicroPython 的智能实体陪伴终端，它不是一个冷冰冰的语音助手，而是一个会随着你的互动而成长的电子生命体。

![壳生同频](docs/status.png)

每一次对话、每一份情绪，都会被它转化为自身属性与状态的变化，沉淀为不可复制的长期记忆。它没有固定人设，所有特质都来自和你的互动——像一个慢慢被你养熟的伙伴，带着你们共同的成长痕迹，陪你走过漫长日常里的细碎瞬间。

---

## 🎯 问题陈述

### 当代陪伴困境

在快节奏的现代生活中，人们面临着前所未有的**情感孤独**与**陪伴缺失**：

1. **传统语音助手的冰冷**：Siri、小爱同学等产品只能完成指令，无法提供真正的情感陪伴
2. **虚拟陪伴的虚无感**：纯软件的 AI 聊天缺乏实体触感，难以建立真实的情感连接
3. **固定人设的局限**：市面上的陪伴机器人都是预设人格，无法真正"懂你"
4. **记忆的易逝性**：每次对话都是全新开始，无法沉淀长期关系

### 核心痛点

- **缺乏成长性**：现有产品无法随用户互动而进化
- **情感单向性**：只能接收指令，无法感知和回应情绪
- **体验碎片化**：软硬件割裂，缺乏一体化的陪伴体验
- **门槛高昂**：高端陪伴机器人动辄数千元，普通人难以负担

## 💡 解决方案

### 核心理念：温养式成长

**壳生同频**提出了革命性的"温养式陪伴"理念：

- **🌱 有机成长**：像养电子宠物一样，通过持续互动让它逐渐形成独特性格
- **🧠 情感记忆**：每次对话都会被转化为长期记忆，形成专属于你们的关系史
- **🎭 动态人格**：没有固定人设，所有特质都来自真实互动
- **💖 实体陪伴**：可爱的小鸡形象 + 触摸屏交互，提供真实的情感寄托

### 产品特色

#### 1. 多模态情感交互
- **语音对话**：基于讯飞中英识别大模型（ASR）+ 语音合成（TTS）的自然对话
- **视觉反馈**：320×240 触摸屏实时显示小鸡动画，6 种状态（待机、倾听、思考、说话、错误、触摸响应）
- **触摸互动**：点击屏幕触发触摸反应动画，增加亲密度，影响性格演变
- **情绪表达**：心形瞳孔、泪滴眼睛、思考气泡等细腻的情感表现

#### 2. 端云协同 AI 架构
- **端侧**：ESP32-S3 运行 MicroPython，实现低延迟的本地交互
- **云侧**：DeepSeek-V3 大模型提供智能对话能力
- **流式响应**：句子级 TTS 合成，边生成边播放，体验流畅自然

#### 3. 低成本高性能
- **硬件成本**：< 200 元（ESP32-S3 开发板 + 触摸屏 + 麦克风 + 扬声器）
- **开发效率**：基于 MicroPython + LVGL，开发周期缩短 70%
- **资源优化**：160×160 ARGB8888 动画仅占用 100KB 内存

## 🛠️ 技术或创意实现

### 技术栈

#### 硬件平台
- **主控**：Waveshare ESP32-S3-Touch-LCD-2（双核 240MHz，8MB PSRAM）
- **显示**：ST7789 320×240 TFT 触摸屏
- **音频输入**：INMP441 MEMS 麦克风（I2S 接口）
- **音频输出**：MAX98357A 数字功放 + 扬声器

#### 软件架构
```
┌─────────────────────────────────────────────────┐
│              用户交互层                          │
│  LVGL 9.3.0 图形界面 + CST816S 触摸驱动         │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│              业务逻辑层                          │
│  • AsyncMicRecorder: 异步录音 + VAD 检测        │
│  • XfyunASR: 讯飞语音识别                       │
│  • XfyunTTS: 讯飞语音合成                       │
│  • OpenAI: 流式对话生成                         │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│              系统层                              │
│  MicroPython v1.25.0 + asyncio 协程调度         │
└─────────────────────────────────────────────────┘
                      ↓
┌─────────────────────────────────────────────────┐
│              硬件抽象层                          │
│  I2S 音频 | SPI 显示 | I2C 触摸 | WiFi 网络     │
└─────────────────────────────────────────────────┘
```

### 核心 AI 能力

#### 1. 智能语音交互
- **ASR**：讯飞中英识别大模型 WebSocket API，16kHz 采样率，支持 202 种方言
- **TTS**：讯飞语音合成（8kHz PCM），流式边合成边播放，减少 1-2s 首字节延迟
- **VAD**：自研语音活动检测，能量阈值 350，连续 40 帧静音触发停止，有效过滤环境噪音

#### 2. 大模型对话
- **模型**：DeepSeek-V3（`deepseek-v3-2-251201`，通过字节跳动 Ark 平台调用）
- **流式生成**：逐句解析 SSE 流，遇到句子结束符（。！？）立即触发 TTS
- **上下文管理**：保留最近 3 轮对话历史，平衡记忆与响应速度
- **情感检测**：识别 5 种情绪，动态调整性格特征值

#### 3. 情感化动画系统
- **状态机设计**：6 种核心状态（IDLE / LISTENING / THINKING / SPEAKING / ERROR / TOUCH）
- **动画引擎**：
  - **待机**：缓慢摇晃浮动 + 随机眨眼
  - **倾听**：兴奋弹跳（1s/周期）+ 音量条动画
  - **思考**：左右摇摆（4s/周期）+ 思考气泡跳动
  - **说话**：心形瞳孔 + 张嘴动画 + 漂浮爱心
  - **错误**：泪滴眼睛 + 紧张颤抖 + 汗滴特效
  - **触摸**：专属触摸反应动画，增加亲密度
- **资源生成**：`gen_assets.py` 使用 PIL 生成 10 帧 ARGB8888 动画（总计 1MB）

#### 4. 情感记忆系统（`memory.py`）

蛋壳拥有持久化的情节记忆，每次对话结束后都会将内容写入 `memory.json`，跨会话保留。

**记忆存储结构：**
```json
{
  "episodes": [
    {"ts": 1747123456, "summary": "今天好累，加班到很晚", "emotion": "tired"}
  ],
  "relationship": {
    "total_rounds": 42,
    "touch_count": 18,
    "closeness": 4
  }
}
```

**工作流程：**

1. 每轮对话结束，调用 `add_episode(user_text, emotion)` —— 取用户输入前 60 字作为摘要，附上情感标签，追加到 `episodes` 列表
2. 列表超过 15 条时，自动丢弃最旧的记录（滑动窗口）
3. 用户触摸屏幕时调用 `add_touch()`，累计触摸次数

**亲密度计算：**

亲密度（0-10 级）由综合分数决定：

```
score = total_rounds + touch_count // 5
```

对应 11 个阈值 `[0, 1, 3, 7, 15, 30, 60, 100, 150, 250, 500]`，分数越高等级越高。对话和触摸都能推进关系，但触摸的权重是对话的 1/5。

**上下文注入：**

每次调用 LLM 前，`build_context()` 将最近 3 条记忆和对话轮数拼成自然语言片段，注入系统提示：

> 我们是老朋友了，已经聊了 42 次，我记得：今天好累，加班到很晚；明天要考试好紧张；刚吃了好好吃的火锅

**动态问候：**

启动时 `greeting()` 根据历史轮数生成不同亲密度的问候语，并附上上次对话的摘要片段，让用户感受到"它还记得你"。

---

#### 5. 动态性格系统（`personality.py`）

蛋壳没有固定人设，性格由 5 个连续特征值（0.0-1.0）描述，随每次对话中检测到的情感实时演变，持久化到 `personality.json`。

**5 维性格特征：**

| 特征 | 初始值 | 激活阈值 | 激活后的表现 |
|------|--------|----------|-------------|
| humor（幽默感） | 0.30 | 0.65 | 喜欢用幽默轻松的方式说话，偶尔说俏皮话 |
| warmth（温柔度） | 0.40 | 0.65 | 非常温柔体贴，总是关心对方的感受 |
| curiosity（好奇心） | 0.30 | 0.65 | 对一切充满好奇，喜欢追问有趣的细节 |
| playfulness（调皮度） | 0.40 | 0.65 | 很调皮可爱，喜欢开小玩笑 |
| empathy（共情力） | 0.30 | 0.65 | 善于感受对方的情绪，能给予恰当的安慰 |

**情感驱动的性格演变：**

每轮对话 LLM 返回情感标签后，调用 `react_to_emotion(emotion)`，对对应特征施加微小增量：

| 检测情感 | 影响特征 |
|----------|---------|
| happy | playfulness +0.02，humor +0.01 |
| sad | empathy +0.03，warmth +0.02 |
| tired | warmth +0.02，empathy +0.01 |
| angry | empathy +0.02，warmth +0.01 |
| anxious | empathy +0.02，warmth +0.02 |
| neutral | curiosity +0.01 |

特征值被 clamp 在 `[0.0, 1.0]`，每 3 轮自动保存一次。

**动态系统提示生成：**

`build_prompt()` 在每次调用 LLM 前执行，只将**已激活**（≥ 0.65）的特征描述拼入系统提示，未激活的特征不出现在提示中。这意味着：

- 刚开始使用时，蛋壳只有 warmth 和 playfulness 接近激活，性格较为平淡
- 长期陪伴后，多个特征被激活，系统提示变得丰富，回复风格明显不同
- 不同用户养出的蛋壳，性格组合完全不同

**完整提示示例（多特征激活后）：**

> 你是蛋壳，一个被主人温养出来的AI电子宠物小鸡。经过相处，你有了自己的性格：你非常温柔体贴，总是关心对方的感受，你很调皮可爱，喜欢开小玩笑，你善于感受对方的情绪，能给予恰当的安慰。我们是老朋友了，已经聊了 42 次，我记得：今天好累，加班到很晚。请简短可爱地回复。
